Pruebas

En la etapa de pruebas consiste en probar el software desde distintos puntos de vistas de una manera organizada y así buscar los posibles errores que se hayan podido cometer en las etapas anteriores para corregirlos antes de que el software sea puesto en producción.

Para ello se pueden realizar varios tipos de pruebas en las que esta:

  • Prueba de unidad:Comprobar el correcto funcionamiento de un componente en particular del sistema.

  • Prueba de integración:Cuando se juntas varios componentes del sistema y sirve para detectar errores en las interfaces.

  • Pruebas Alfa:Pruebas realizadas por el área de desarrollo desde el punto de vista de un usuario final, que ayudan a pulir los aspectos de interfaz de usuario.

  • Pruebas Beta:Pruebas realizadas por saurios finales ajenos al equipo de desarrollo y son vitales para el éxito del proyecto.

Con la aceptación por parte del cliente se da por terminado el proyecto.
